All the important variables required for modelling are present in a variable of struct type `state`. It contains information like the current mode, coordinates of all points and their corresponding colors, centroid coordinates.

A `zpos` variable is used which stores the current z-coordinate in modelling mode. It can be changed using Up(+) and Down(-) arrow keys in modelling mode, allowing the artist to draw in 3D. All other controls are as given in the specification.

The VBO stores the coordinates of points followed by the color vectors. The coordinates and color and the transformation matrix is passed to the vertex shader which sets up gl_Position and passes on the colors to the fragment shader. The transformation matrix for modelling mode is always identity, while for inspection mode on can move around through translation and rotation in accordance with the specification.

Modelling mode always shows the plotted points, to see the current figure one has to use the inspection mode. Colors have to be added in the file from which the model is loaded, otherwise points have a default color.

The model can be saved to a file with the K key, then we can continue to add points. We can load a model back from a file, inspect it (in the inspection mode) and continue to add points to it (in the modeling mode).

You can use clear all the points using C key.

We have assumed that when we load a model from a file, the points we have added earlier are removed.

Also, we’ve interpreted the specification "Every three successive vertices should form a triangle." to mean that if we have points 1,2,3,4,5,6 the triangles would be 123, 234, 345, 456, thus allowing continuity in the model. Our models have been created keeping this in mind.
